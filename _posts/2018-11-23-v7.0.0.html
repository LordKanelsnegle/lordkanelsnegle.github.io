New Features:

  • Migrated to .NET Core! After over a month of downtime due to this migration process (largely because of issues with preserving data), Cinnamon is finally back up and running :D
  • Bot's status will be set to idle while rebooting so that users know the bot will not respond

Bug Fixes and Adjustments:

  • Completely reworked the way users are handled by the database, exponentially improving boot time and making it infinitely faster (literally)
  • Reduced the downtime between updates so that Cinnamon will now only be down for a minute or so while updating
  • Updated the way timestamps are displayed when using the userinfo command for consistency
  • The "Favourite Command" section of the userinfo command now also displays the command's group (e.g. "cah join" instead of just "join")
  • Vastly optimised the Battle MMR algorithm and gains and losses are now applied regardless of the MMR difference of the players, firstly because it was a poor attempt at preventing MMR grinding and secondly because it caused confusion
  • The final screen of a Battle now displays the players in order of points from highest to lowest, instead of in order of when they joined the lobby, to clarify who obtained which position (1st, 2nd, 3rd, etc)
  • Edited userinfo and the "... is the deathbattle champion" status so that both use "Battle" instead of "deathbattle" for consistency
  • Corrected the credits command to only display credits for the new joined website instead of the old, individual changelog and commands websites
  • Fixed a bug that caused gaydar and ship bars to be filled incorrectly
  • gaydar and ship percentages no longer have a leading 0 when below 10%
  • gaydar and ship display bars are now spaced correctly
  • Updated the website, removing the 'Changelog' tab from navigation and adding the 'Donate' tab
  • Website music is now off by default
  • Unknown error message now has a facepalming panda
  • Sprinkled with love and affection
