New Features:

  • Users will now be notified when one of their actions has caused an exception to avoid people repeatedly trying to execute the same bot-breaking code multiple times
  • All Discord requests will now automatically retry if they failed, which should increase bot reliability and reduce the number of exceptions/errors the bot encounters
  • There is now a {user} is my best friend :3 status, where {user} is set to the user that has used the largest number of commands
  • Completely redigned discordbot websites to make them smexy (bots.discord.by and discordbots.org)

Bug Fixes and Adjustments:

  • Fixed a problem with timed autoroles and reminders that caused the bot to crash for durations longer than 24 days, largely increasing performance in the process
  • Removed a chunk of autorole code that turned out to be completely useless (must’ve been written at 2am)
  • Split the message handler in half so that autoroles and guildstats are handled separately, hopefully this should reduce the amount of gateway blocking that occurs
  • Enhanced the way connectfour handles reactions being added, increasing performance significantly
  • Reworked the battle command from the ground up - it now behaves like the rest of the games, with similar commands and display formats as well as improved efficiency/reliability/readability
  • Cleaned up other game formats so that they all have a consistent format
  • Bot will no longer send annoying “Commands cannot be used in this channel” on the support server (wont affect most people)
  • Statuses involving users will no longer include me as I’d like them to reflect the community :3
  • Sprinkled with love and affection
  • Fixed a problem that caused reminders to trigger multiple times as they weren't being cleared from the database
  • Cinnamon should now properly DM exceptions to me, as intended from a few patches ago, along with the message context and other debugging data
  • Resolved an issue that prevented users from joining cah games which were already in progress
  • Fixed a bug that caused the bot to throw an exception if a second cah card was picked with a numeric value equal to the number of cards a user has
  • Flattened out a problem in which the bot would throw an exception if it attempted to delete a non-existent message
  • connectfour board embeds will now display "Gameover! {user} won." in the footer
  • This change doesn't affect users, but the way exceptions are handled and logged has now been changed to clutter/spam the log less
  • Sprinkled with love and affection
