New Features:

  • Added log and update commands (for my personal use only)
  • Added credits command which will display credits for things like game concepts, art, and suggestions - this will be updated as the bot is developed further
  • userinfo now also displays the user's most frequently used command
  • Added stat commands which display analytical data such as most active days, most active hours, most active users, and much more that took TONS of time to work out. You can use guildstats, channelstats, or userstatsยน depending on which kind of stats you want. Note that all stats are averages, recorded from the moment this update goes live (or the moment the bot joins the server), so it will only be an approximation and will not reflect data from the past

Bug Fixes and Adjustments:

  • Fixed a bug in which the active user count was (wrongly) being set to a value larger than the encountered user count
  • Completely revamped a large portion of the code to utilise timers, largely reducing the stress on threads
  • Revamped userinfo command so that it looks a bit nicer - Note that it will most likely be reworked completely in the near future
  • Altered the way the permissions system works so that the bot only requests permissions it needs if it doesn't have it for a particular command. For example, if it doesn't have the ManageMessages permission it will only refuse to execute commands which need it (instead of rejecting all commands, as it did before)
  • I will now be personally notified, via DM, every time the bot encounters an exception (runs into a bug) so that I can patch problems as fast as possible :3
  • Fixed a bug causing the bot to crash far more frequently than it needed to
  • Paged embeds (multipage embeds that allow users to change pages using reactions) now display the reason why they close instead of just deleting themselves (e.g. if a user terminates it or if it times out)
  • Resolved yet another issue with broadcast messages
  • Commands will now fail if the bot doesn't have the Embed Links permission, as they should
  • Greatly enhanced database robustness and patched a bug that allowed faulty channels to be stored
  • Modified the birthday feature so that users are now wished a happy birthday not only via DM but also in every guild they share with the bot modchannel, logchannel, and cmdchannel commands can now only be used on channels which the bot has permissions in
  • Numerous other bug fixes and efficiency improvements
  • Sprinkled with love and affection
