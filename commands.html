Help!

So you've stumbled across Cinnamon but haven't the faintest clue how to use him. Or perhaps you've just forgotten one or two commands and how to properly use them. Heck, maybe you're just here because you've got nothing better to be doing. Either way, you'll find a full list of every command, its aliases, description, and example usage below :3

Please note that Cinnamon is still quite far from completion and as such it's possible (though unlikely) that you may run into bugs. If you are ever in a position where Cinnamon displayed unexpected behaviour such as failing to respond to a command, please let me know instead of simply kicking him so that I can fix the problem.

Syntax

There are some sub-commands with the alias ' ' - this means that if nothing is written after the parent command, this command will be executed by default. For example, c!jm list and simply c!jm will both list the server's welcome messages. When using a role, channel, or user as an argument you can use their ID, tag, or name.
All textual arguments used in a command (except the last one) must be enclosed with quotation marks if they contain spaces (shown below).
c!ship "Lord Kanelsnegle" John Cena

Configuration

The following commands are used to configure the bot to your liking so that it can suit your server's needs as perfectly as possible. Every single one of these commands can ONLY be used by the server owner or Cinnamon moderators (the role for this can be set using the `modrole` command).

prefix

Set the bot’s command prefix, no longer than 10 characters.
Usage: prefix <prefix...>

deletemsg

Toggles the deletemsg configuration option, which is disabled by default. When enabled all command invoking messages will be deleted once the command executes.
Usage: deletemsg

msgchannel

Set the channel that you want messages to be sent to when users join or leave the server, as well as information from the latest patch and any important notices. By default the bot will automatically set this to your guild’s welcome channel - if there isn’t one set then it’ll be set to the first channel found with the word “announcement”. If the bot still can’t find a default channel the message channel will be set to the first channel which the bot can send messages to. Set this to off or disable to disable messages all together, though it’s bly recommended that you keep notifications enabled in case of any major changes to Cinnamon’s functionality.
Usage: msgchannel <textChannel...>|off|disable

add, +, create

Add a text channel to your guild’s list of command channels. If your guild has any command channels then Cinnamon’s command usage will be restricted to those channels.
Usage: cmdchannel add <textChannel...>

delete, del, -, remove

Remove a text channel from the list of command channels set for your guild.
Usage: cmdchannel delete <textChannel...>

list, ' '

Lists all of the currently set command channels for your guild.
Usage: cmdchannel list

clear, clr

Remove all of the command channels set for your guild.
Usage: cmdchannel clear

add, +, create

Add a message to the list of welcome messages that are used to welcome users to your guild when they join; if there is more than one welcome message set, the bot will pick one at random when a user joins. By default the user will be mentioned at the beginning of the message, but you can use {user} at any point in your message if you would like to move where the mention is, e.g. joinmsg add Welcome {user}! will output Welcome @Username! when a user joins. There is a limit of 15 welcome messages per guild.
Usage: joinmsg add <message...>

delete, del, -, remove

Remove a message from the list of welcome messages that are used to welcome users to your guild when they join. To get the number/position of the message you want to remove, use the joinmsg list command.
Usage: joinmsg delete <numericPosition>

list, ' '

Lists all of the currently existing welcome messages for your guild alongside their respective positions in the list. Use the reactions added to the message to flip through the pages.
Usage: joinmsg list

clear, clr

Remove all of the welcome messages set for your guild.
Usage: joinmsg clear

add, +, create

Add a message to the list of farewell messages that are used to say goodbye to users in your guild when they leave; if there is more than one farewell message set, the bot will pick one at random when a user leaves. By default the user’s name will be at the beginning of the message, but you can use {user} at any point in your message if you would like to move where their name is, e.g. leavemsg add Farewell {user}! will output Farewell Username#1234! when a user leaves. There is a limit of 15 farewell messages per guild.
Usage: leavemsg add <message...>

delete, del, -, remove

Remove a message from the list of farewell messages that are used to say goodbye to users in your guild when they leave. To get the number/position of the message you want to remove, use the leavemsg list command.
Usage: leavemsg delete <numericPosition>

list, ' '

Lists all of the currently existing farewell messages for your guild alongside their respective positions in the list. Use the reactions added to the message to flip through the pages.
Usage: leavemsg list

clear, clr

Remove all of the farewell messages set for your guild.
Usage: leavemsg clear

add, +, create

Add a role to your guild’s list of automated roles, with an optional condition.

A timespan can be included so that users are only given the role after they’ve been in the server for a certain amount of time - e.g. autorole add Role 10h will add the role “Role” to users that have been in the server for 10 hours or longer. The bot will only accept timespans in terms of weeks, days, hours, minutes, and seconds.

Likewise, a message count can be included so that users are only given the role after they’ve sent a certain amount of messages - e.g. autorole add Role 10 messages will add the role “Role” to users that have sent 10 or more messages. When using this condition you MUST include the word “message” or “msg” to specify that it is a message condition and not a timespan.

If no timespan or message number is supplied then the role will be immediately given to new users as soon as they join the guild.
Usage: autorole add <role...> [timespan|messageCount]

delete, del, -, remove

Remove a role from the list of automated roles set for your guild, preventing it from being added to future users. You can optionally add the word fully to the end to additionally remove the role from all current users.
Usage: autorole delete <role...>
Alternate Usage: autorole delete <role...> fully

list, ' '

Lists all of the currently set automated roles for your guild.
Usage: autorole list

clear, clr

Remove all of the automated roles set for your guild.
Usage: autorole clear

Info

This group of commands gives you any information you may require, whether about the bot, commands, or scoreboards.

help

Provides useful links such as the one that probably sent you here. Providing a command as an argument will link you directly to that command's position on this page.
Usage: help
Alternate Usage: help <commandName>

info, i, stats

Display information about the bot.
Usage: info

configuration, config, conf

Displays the bot’s current configuration.
Usage: configuration

ping

Returns the bot’s latency. THE BOT’S LATENCY.
Usage: ping

userinfo, user info, ui, whois, show, profile, user

Displays the profile of a given user, or yours if no user is specified
Usage: userinfo [user...]

guildinfo, guild info, gi, guild, serverinfo, server info, si, server

Retrieves the guild’s current information, including user count and date of creation.
Usage: guildinfo

guildstats, guild stats, gs, serverstats, server stats, ss

Retrieves the guild’s current statistics, containing records such as the longest vc and the most users in a vc at once as well as analytics like the average guild activity per-hour and per-day.
Usage: guildstats

credits

Displays credits for things like game concepts, art, and suggestions - this will be updated as the bot is developed further.
Usage: credits

setup, create

Sets up a new CAH lobby.
Usage: cah setup

cancel

Cancels the lobby. This command is restricted to the Game Creator and server admin.
Usage: cah cancel

join

Joins an open lobby - alternatively, if there is a running CAH game in which users are allowed to join while the game is in progress, the user will join that game and be dealt their cards at the end of the current turn.
Usage: cah join

leave, quit

Leave the game, if you are in one.
Usage: cah leave

kick, remove

Remove a user from the CAH game or lobby. Please note that once removed a user cannot rejoin the CAH game and must wait for the game to end to play in that channel. This command is restricted to the Game Creator and server admin.
Usage: cah remove <user...>

start

Starts the game if at least 3 people have joined. This command is restricted to the Game Creator and server admin.
Usage: cah start

end

Ends the game prematurely. This command is restricted to the Game Creator and server admin.
Usage: cah end

info, ' '

Provides some information about Cards Against Humanity and the configuration options.
Usage: cah info

pick

Pick cards from your hand to fill in the blanks given. If there are multiple blanks you can pick multiple cards by separating them with spaces. If blank cards are enabled and you pick one or more blank cards, you will be prompted to enter custom phrases via DM - if you fail to do so by the end of the round your blank cards will be played as “(Blank)”. Alternatively, if you are the Card Czar, this command will allow you to pick the winning card(s) played.
Usage: cah pick <cardNumber> [cardNumber] [cardNumber]

configure, config, set

Allows you to configure the game before starting it. Sub-Commands:

  • dropin: Toggles dropin, allowing users to join while the game is in progress if enabled.
  • blankcards: Sets the number of blank cards in the deck, no less than 0 and no more than 100.
  • timeout: Sets the number of seconds to wait before skipping a player, no less than 60 and no more than 300.
  • maxscore: Sets the score needed to win the game, no less than 5 and no more than 30.
  • maxplayers: Sets the maximum number of players that can join the game, no less than 3 and no more than 20.

Usage: cah configure dropin
Alternate Usage: cah configure blankcards|timeout|maxscore|maxplayers <value>

Other

Other, miscellaneous commands which do not fall into the previous categories or exist just for fun are listed below.

quote, “, qu

Quote a message. Alternatively, create a fake quote.
Usage: quote <messageID>
Alternate Usage: quote <user> <text...>

avatar, av, profile, icon

Displays an enlarged version of a given user’s avatar, or yours if no user is specified.
Usage: avatar [user]

birthday, bday

Set your birthday. This will be displayed on your profile and will make the bot wish you a happy birthday on your birthday as well as gifting you 500 tokens. For internet safety reasons, only the day and month are required. Please use a number for the day but a word for the month (e.g. 22 Jan or 22 January).
Usage: birthday <day> <month>

bio, description, desc

Sets your bio, which is displayed on your profile. Your bio can take up multiple lines if you use multiple lines when you input it, e.g.
~bio so
many
lines!
will show as multiple lines on your profile as well.
Usage: bio <description...>

daily

Grants either you or a given user anywhere from 100 to 300 tokens, excluding streak bonuses for using this command every day. Refreshes every 24 hours.
Usage: daily [user]

n00dles, noodles, n00dle, noodle, nudes, n00ds, noods

Sends a random picture of noodles.
Usage: n00dles

dab

Sends a random dabbing gif.
Usage: dab

gaydar

Uses the gaydar to determine how gay a given user is, or yourself if no user is specified.
Usage: gaydar [user]

dicklength, dlength, dl

Find how long a given user’s dick is, or your own if no user is specified.
Usage: dicklength [user]

grouphug

Initiate a group hug which people can join by adding a hug reaction. The more users join, the more people will be in the group hug gif sent after 5 seconds.
Usage: grouphug

waifurate, wr

Gets the bot to rate how much of a waifu a given user is, or yourself if no user is specified.
Usage: waifurate [user]

insult, roast

Make the bot insult a given user or yourself if no user is specified.
Usage: insult [user]

ship

If two users are provided the bot will determine how successful their relationship is and generate as many shipnames as it can using their usernames. Alternatively, two names can be provided (as text) and the bot will just generate shipnames using those two names.
Usage: ship <user> <user2>
Alternate Usage: ship <name> <name2>

remindme, setreminder

Set a reminder. The timespan can be any mixture of days, hours, minutes, and seconds in any order (e.g. “5h 10s” or “78 min and 3h”)
Usage: remindme <message> in <timespan>

mood

Ask Cinnamon how he’s feeling today :3
Usage: mood
